菜鸟级的21年前端技术学习规划.md

Created2021-05-31|Updated2021-06-01
|Post View:

搬砖人员的业绩贡献

  1. 前端基建，工程化配置，文档整理，开发部署流程上的改进
  2. 需求评审与方案评审
  3. 代码的规范，健壮，易于维护，
  4. 不仅仅是个搬砖人员，利用技术能力保障业务的稳定性，持续推进，对齐业务指标

而菜鸟想要晋级就需要时间，需要日益渐进，需要持续学习与大量的思考
更需要一番热情，去主动探索与实践。

给自己两年时间，你准备如何成长

  1. 阅读源码
  2. 参加开源项目
  3. 对于业务的思考，业务困难驱动造轮子,很多技术上的想法，都来自业务本身遇到的问题
  4. 尽量保证一两周一次博客

两年后，你准备成长到什么地步,你能承担多大的责任

  1. 心理心态
  2. 技术上
  3. 业务上

技术学习规划

工程化配置

  1. cli搭建工具
  2. 打包构建
  3. webpack vite
  4. 部署CI/CD，pipeline，workflow

代码规范

  1. eslint
  2. prettier
  3. sonarjs
  4. JavaScript standard 代码规范

vue，react源码对比深入解读

  1. vue,react的虚拟dom有什么不同，新旧节点的对比
  2. fiber架构子任务优先级的思想
  3. 状态管理： 对比vuex,redux(单向数据流),mobx(响应式数据流)，RXjs(stream)
  4. 函数式组件(react hooks)与class类组件
  5. umi与dva

性能优化

  1. Lighthouse，Performance报告
  2. 浏览器网络
  3. FMP计算
  4. Vue-react性能优化
  5. 开源项目tools

搭建一个自己的组件库

  1. 拖拽
    • 穿梭框 + 拖拽
    • 列表上下拖动
  2. 滚动，基于better-scroll

微前端架构

  1. qiankun，基于single-spa

    多个 Bundle 如何集成？
    子应用之间怎样隔离影响？
    公共资源如何复用？
    子应用间怎样通信？
    如何测试？

大前端，移动端开发

  1. 原生native
  2. hybrid混合开发
  3. h5
  4. react native
  5. flutter
  6. 北海kraken渲染引擎

参加开源项目

目前还在找，并未落实

算法与数据结构 LeetCode

  1. 链表
  2. 动态规划dp
    • 爬楼梯
    • 打家劫舍

  3. 背包
  4. 多多益善
